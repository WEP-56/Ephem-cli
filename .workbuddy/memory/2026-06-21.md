# 2026-06-21 工作日志

## Ephem 项目开发（首次搭建）

实现了"临时、端到端加密命令行聊天室"的完整后端 + 可用 CLI，位于 `E:\ephem-cli`。

### 技术栈与结构
- 后端：Cloudflare Workers + Durable Objects（`packages/backend-cloudflare`）
- CLI：Node + ink + ws（`packages/cli`），命令名 `ephem`
- 两个包独立安装（npm workspaces 在 Windows 符号链接失败 → 改为各包独立 `npm install`，根 package.json 用 `--prefix` 脚本串联）

### 关键实现决策
- **房间码**：BIP39 英文词表(2048词)随机抽3词，如 `correct-horse-battery`；后端只存 SHA-256 哈希，DO name = 哈希（无明文）。
- **加密**：HKDF(SHA-256) 从房间码派生 32 字节密钥 → AES-256-GCM，authTag 拼在密文末尾，base64 传输。客户端本地完成，后端只转发密文。
- **RoomObject DO**：内存 sessions Map + storage 存 meta；alarm 实现 TTL 自毁 + 空房间 5 分钟宽限；单 IP 限流(15次/分钟)；人数上限校验。
- **workerder 关键坑**：accept() 后立即 send 首条 joined 消息，在并发连接时会丢失。**修复：用 `setTimeout(0)` 把首条 joined/peer_joined 延迟到 Response 返回后再发**。这是本地联调时发现的。
- Admin 页面：单页 HTML+JS，`[assets]` 自动托管 `public/`，`X-Admin-Key` 鉴权。
